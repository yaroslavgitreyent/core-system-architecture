# Розділ 10. Аналіз системи безпеки інформаційної платформи

## 10.1. Стратегія багаторівневого захисту
Для системи "EcoSmart City" безпека є критичним пріоритетом, оскільки маніпуляція екологічними даними може призвести до паніки або приховування техногенних катастроф. Захист реалізовано на трьох рівнях:

### 1. Безпека передачі даних (Network Security)
* **Шифрування TLS/SSL:** Весь трафік між датчиками, сервером та клієнтами шифрується за протоколами HTTPS та MQTTS. Це унеможливлює атаки типу "Man-in-the-Middle" (MitM).
* **Захист від DDoS:** Використання хмарних фільтрів (наприклад, Cloudflare) для відсікання аномального трафіку, що намагається перевантажити API.

### 2. Контроль доступу (Access Control)
* **Рольова модель (RBAC):** Чітке розмежування прав. Мешканці мають доступ лише до читання, екологи — до аналітики, а адміністратори — до конфігурацій.
* **Автентифікація через JWT:** Використання токенів JSON Web Tokens із коротким терміном дії та механізмом Refresh-токенів для безпечного входу в мобільний додаток.

### 3. Безпека IoT-пристроїв
* **Унікальні сертифікати:** Кожен датчик має унікальний ID та сертифікат безпеки. Система ігнорує дані від неавторизованих або підроблених пристроїв.
* **Обмеження запитів (Rate Limiting):** Захист від спаму: якщо датчик починає надсилати дані занадто часто, він тимчасово блокується для аналізу на предмет зламу.

## 10.2. Захист конфіденційності (GDPR)
Система відповідає вимогам захисту персональних даних. Геолокація користувачів зберігається у знеособленому вигляді, а персональна інформація (email, паролі) шифрується алгоритмом хешування Argon2 або bcrypt.

# Практична робота №3. Налаштування SSH-ключів та безпека доступу

## 1. Обґрунтування використання SSH
Для взаємодії з віддаленим репозиторієм GitHub було обрано протокол SSH (Secure Shell). Це дозволяє забезпечити:
* Шифрування даних під час передачі.
* Авторизацію без необхідності введення логіна та пароля при кожній операції.
* Підвищений рівень безпеки за рахунок використання пари ключів (приватного та публічного).

## 2. Технічний процес налаштування
Процес налаштування складався з наступних етапів:
1.  **Генерація ключа:** Використано алгоритм Ed25519, який забезпечує найкраще співвідношення швидкості та безпеки. Команда: `ssh-keygen -t ed25519 -C "sevacenkokuku@gmail.com"`.
2.  **Додавання до SSH-агента:** Налаштування локального середовища для автоматичного керування ключами.
3.  **Реєстрація на GitHub:** Публічний ключ був доданий у налаштування профілю користувача `yaroslavgitreyent`.
4.  **Валідація з'єднання:** Виконано тестування через команду `ssh -T git@github.com`, що підтвердило успішну автентифікацію.

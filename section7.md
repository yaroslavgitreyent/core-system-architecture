# Розділ 7. Побудова моделі даних

## 7.1. Концептуальна модель даних
Для функціонування системи розроблена реляційна модель, що описує взаємозв'язки між фізичними пристроями, вимірюваннями та користувачами.

## 7.2. Опис основних таблиць та атрибутів

### 1. Таблиця "Користувачі" (Users)
- **ID:** Унікальний ідентифікатор (Primary Key).
- **Username/Email:** Облікові дані для входу.
- **Role:** Роль користувача (admin, ecologist, resident), що визначає права доступу.

### 2. Таблиця "Пристрої" (Devices)
- **ID:** Унікальний номер датчика.
- **Type:** Тип сенсора (газовий, шумовий, пиловий).
- **Coordinates:** Географічне розташування (Latitude, Longitude).
- **Status:** Стан пристрою (активний, потребує обслуговування, вимкнений).

### 3. Таблиця "Вимірювання" (Measurements)
- **ID:** Ідентифікатор запису.
- **Device_ID:** Посилання на пристрій (Foreign Key).
- **Value:** Числове значення показника.
- **Timestamp:** Точний час та дата заміру.
- **Unit:** Одиниці виміру (наприклад, мкг/м3 або дБ).

### 4. Таблиця "Сповіщення" (Notifications)
- **ID:** Ідентифікатор повідомлення.
- **User_ID:** Кому надіслано (Foreign Key).
- **Type:** Рівень критичності (Warning, Critical).
- **Message:** Текст сповіщення.

## 7.3. Зв'язки між сутностями
- Один **Користувач** може отримувати багато **Сповіщень** (1:N).
- Один **Пристрій** може генерувати тисячі **Вимірювань** (1:N).
- Зв'язок між **Користувачем** та **Пристроєм** здійснюється опосередковано через географічну прив'язку вимірювань до локації користувача.

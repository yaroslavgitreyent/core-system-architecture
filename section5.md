# Розділ 5. Візуалізація функціональних сценаріїв у UML-нотації

## 5.1. Призначення діаграми прецедентів
Для формалізації взаємодії між користувачами та інформаційною системою "EcoSmart City" було використано діаграму прецедентів (Use Case Diagram) у нотації UML. Мета цієї діаграми — зафіксувати функціональні межі системи та визначити, які саме ролі (актори) ініціюють конкретні сценарії використання.

## 5.2. Графічне представлення (UML Use Case)

```mermaid
graph LR
    User((Мешканець міста))
    Eco((Еколог-аналітик))
    Admin((Адміністратор))
    Sensor((IoT-датчик))

    subgraph "Платформа EcoSmart City"
        UC1(Моніторинг карти)
        UC2(Налаштування сповіщень)
        UC3(Аналіз трендів)
        UC4(Генерація звітів)
        UC5(Керування пристроями)
        UC6(Передача телеметрії)
    end

    User --> UC1
    User --> UC2
    Eco --> UC1
    Eco --> UC3
    Eco --> UC4
    Admin --> UC5
    Sensor --> UC6

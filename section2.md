# Розділ 2. Збір, аналіз та специфікація вимог до ІС "EcoSmart City"

## 2.1. Вступ та методологія аналізу
Для забезпечення високої якості системи "EcoSmart City" було проведено комплексний аналіз потреб стейкхолдерів. Основний фокус спрямований на інтеграцію великих масивів даних (Big Data) з мережі IoT-сенсорів та забезпечення оперативного доступу до них для кінцевих користувачів.

## 2.2. Функціональні вимоги (Functional Requirements)
Система повинна виконувати наступні ключові функції:
1. **Збір даних у реальному часі:** Підтримка протоколів передачі даних MQTT та HTTP для отримання показників від мережі датчиків.
2. **Валідація та очищення:** Автоматична фільтрація аномальних значень (шумів), які можуть виникати при несправності фізичних сенсорів.
3. **ГЕО-візуалізація:** Інтеграція з картографічними сервісами (OpenStreetMap/Google Maps) для відображення шарів забруднення повітря та шуму.
4. **Система сповіщень:** Модуль автоматичної розсилки push-повідомлень та email при досягненні критичних показників (норми PM2.5, CO2).
5. **Генерація звітності:** Можливість агрегації даних за довільний період (день, тиждень, місяць) з виведенням графіків та статистичних показників у форматах PDF/XLSX.

## 2.3. Нефункціональні вимоги (Non-Functional Requirements)
* **Надійність (Reliability):** Коефіцієнт готовності системи має становити не менше 0.995. Система повинна мати механізм автоматичного відновлення після збоїв.
* **Продуктивність (Efficiency):** Обробка пакету даних від 1000 датчиків одночасно не повинна перевищувати 2 секунди.
* **Безпека (Security):** Використання протоколу HTTPS для веб-доступу та TLS для передачі даних з датчиків. Шифрування персональних даних користувачів.
* **Локалізація:** Підтримка багатомовності інтерфейсу (українська, англійська).
